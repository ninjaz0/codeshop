# 🚀 Магазин виртуальных товаров (Codeshop)

> **Примечание: Этот проект был полностью разработан и создан с помощью ИИ.**

Это легкая, современная платформа для автоматической выдачи и покупки виртуальных товаров. Используя архитектуру с разделением фронтенда и бэкенда, система фокусируется на автоматической выдаче и процессах транзакций виртуальных предметов (таких как ключи активации программного обеспечения, купоны на членство, текстовые руководства и т.д.). Пользователям не нужно платить реальными деньгами; вместо этого они могут увеличить баланс своего аккаунта, погасив "Коды активации", сгенерированные администратором, и использовать этот баланс для покупки цифровых товаров.

---

## ✨ Основные функции

- **🌍 Полная поддержка мультиязычности (i18n)**
  - Система имеет встроенную и полную адаптацию для 5 языков: упрощенный китайский, традиционный китайский, английский, японский и русский.
  - Пользователи могут свободно переключать языки на главной странице в любое время.

- **💳 Замкнутая виртуальная экономика**
  - **Система пополнения**: Администраторы генерируют "Коды активации" с определенным номиналом в панели управления.
  - **Использование баланса**: Пользователи вводят код для пополнения баланса и используют его для покупок в магазине.

- **📦 Умная система автоматической выдачи**
  - **Режим «Уникальный ключ» (Unique Key)**: Один ключ на одну покупку. При каждой продаже система автоматически списывает товар со склада и выдает пользователю уникальный ключ.
  - **Режим «Фиксированный контент» (Fixed Content)**: Бесконечный запас. Все пользователи, покупающие этот продукт, получат одну и ту же ссылку на скачивание или фиксированный текст.

- **🔐 Строгая система инвайтов и безопасности**
  - **Регистрация по инвайтам**: Регистрация новых пользователей требует "Код приглашения" (инвайт), сгенерированный администратором, что эффективно контролирует злоупотребления.
  - **Защита данных транзакций**: Списания средств, учет остатков и использование кодов активации полностью защищены транзакциями базы данных SQLite, предотвращая одновременные перепродажи и аномалии данных.

- **🛠️ Мощная панель управления (Admin Dashboard)**
  - Управление публикацией товаров и импорт запасов уникальных ключей.
  - Массовая генерация и копирование кодов пополнения и инвайтов в один клик.
  - Список зарегистрированных пользователей, изменение баланса, управление ролями и защита администратора от случайного удаления себя.

## 🛠️ Технологический стек

- **Фронтенд**: Vue 3 + Vite + Vue Router + Pinia + Vue-i18n
- **Бэкенд**: Node.js + Express + JWT Аутентификация
- **База данных**: SQLite3 (Использование `better-sqlite3` для высокопроизводительных синхронных запросов и поддержки транзакций)

## 🚀 Развертывание в один клик

Проект включает в себя готовый скрипт для развертывания, который автоматически проверяет окружение, устанавливает зависимости, компилирует фронтенд (Vue) и запускает бэкенд-сервис Node.js через демона PM2 (поскольку бэкенд настроен на отдачу статических файлов, установка Nginx не требуется).

**Системные требования**: Сервер на Linux / macOS с установленным Node.js (`v16+`).

### 1. Клонирование и переход в директорию
```bash
git clone https://github.com/ninjaz0/codeshop.git
cd codeshop
```

### 2. Запуск скрипта развертывания
```bash
chmod +x deploy.sh
./deploy.sh
```

> **Готово!** После успешного развертывания сервис будет доступен по умолчанию по адресу `http://127.0.0.1:3000`. Вы можете получить доступ к сервису, введя IP-адрес вашего сервера и этот порт в браузере.

При первом запуске системы и инициализации базы данных автоматически создается учетная запись администратора с высшими правами.

- **Логин**: `admin`
- **Пароль**: `admin123`

*(⚠️ Настоятельно рекомендуется перейти в «Настройки» в правом верхнем углу и изменить пароль по умолчанию после первого входа.)*

## 📄 Полезные команды (на базе PM2)

Скрипт автоматически устанавливает глобальный `pm2` для управления бэкенд-процессом:

- **Проверить статус сервиса**: `pm2 status`
- **Просмотр логов в реальном времени**: `pm2 logs virtual-mall`
- **Перезапустить сервис**: `pm2 restart virtual-mall`
- **Остановить сервис**: `pm2 stop virtual-mall`
